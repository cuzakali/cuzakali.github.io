    这篇文章主要是用来记录下，我挖洞的思路以及挖洞的手法，由钱入深细细道来

    • 目标
      • 中国邮政
      • 铁路公司

    涉及一些关键东西我可能会作打码处理，请见谅

    信息收集

    ** 首先我拿到了我要检测的域名，下面简称资产，哇好多的感觉，￥￥￥￥￥￥ - -,然后我们进行第一步，扫子域名咯，在就是语法收集一波咯，这里我们需要判断的有**

    • 是否采用了cdn。
      • 启用了什么端口。
    • 采用的是Windows还是Linux系统。
    • 用的什么程序，使用的什么语言，容器是什么。
      • 网站是否开启了伪静态
      • 能否查询到cms指纹
    • 是否采用了waf，采用的什么waf。
      • 网站是否采用了加密
    • 然后语法收集一波pdf,doc,docx,xls,ttf，,rar,zip,等文件
      • 判断有没有什么有用的信息，保存下来。
    • 因为考虑到不让扫目录，所以需要语法中搜索一下
      • 比如admin login password registered
      • 在就是文件上传 、手机号、用户名、密码、身份证、使用方法、登录等字符搜索，在有最关键的一步，我建议把网站公开的邮箱和姓名给收集起来，搞不好有大用。
    • 业务逻辑：比如注册的逻辑是什么？找回密码的逻辑是什么？购买的逻辑是什么，优惠券购物车等逻辑，逻辑有了之后；
      • 在进行抓包判断有没有什么关键接口泄漏
      • 这一步可以判断出post提交参数有没有加密，存不存在爆破，有没有注入或者xss等，当然最好是留在最后，因为需要判断有没有什么waf，在综合信息。
    • 之后就是总结咯，把信息综合起来，对照我上面说的一样一样进行完毕之后，你会得到一大波庞杂的信息，然后就是分析咯。
